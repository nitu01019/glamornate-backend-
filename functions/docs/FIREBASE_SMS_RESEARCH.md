# Firebase SMS Research — Definitive Answer
_Researched: 2026-04-27 | Author: firebase-master (glamornate-master-audit team)_

---

## Section 1 — Definitive Answer

**Firebase has NO inbuilt API for sending arbitrary transactional SMS.** Every Firebase product that involves SMS (Phone Auth, Identity Platform MFA) is strictly restricted to sending one-time verification codes for authentication purposes only. Sending custom text like "Your therapist is on the way" requires a third-party SMS carrier — available either as a Firebase Extension (wrapping that carrier) or as a direct SDK call from Cloud Functions.

## Section 3 — Firebase Extensions Marketplace: SMS Extensions

### (a) Officially Firebase/Google-Maintained Extensions

None. There is no SMS extension authored or maintained by Firebase/Google directly.

> The "Trigger Email from Firestore" extension (`firebase/firestore-send-email`) is the only messaging-category extension maintained by Firebase. It sends email, not SMS.

### (b) Community-Maintained Extensions Distributed via Firebase Extensions Hub (extensions.dev)

All SMS extensions on extensions.dev are maintained by their respective third-party companies, not by Firebase:

| Extension | Publisher | Carrier | URL |
|---|---|---|---|
| Send Messages with Twilio | Twilio Labs (twilio-labs GitHub org) | Twilio Programmable Messaging API | https://extensions.dev/extensions/twilio/send-message |
| Send Messages with MSG91 | MSG91 | MSG91 API | https://extensions.dev/extensions/msg91/msg91-send-msg |
| Send Messages with MessageBird | MessageBird | MessageBird Conversations API | https://extensions.dev/extensions/messagebird/firestore-messagebird-send-msg |
| Vonage Messages API | Vonage | Vonage/Nexmo API | https://extensions.dev/extensions/vonage/firebase-vonage-messages |
| Send Message with Nimba SMS | Nimba SMS | Nimba SMS API | https://extensions.dev/extensions/nimbasmsapi/nimbasmsapi |

**Note on Twilio:** Twilio is often cited as having "partnered with Firebase" because Twilio Labs published their extension specifically for the Firebase Extensions Hub. However, the publisher in `extensions.dev` is `twilio` (not `firebase`), the GitHub repo is `twilio-labs/twilio-firebase-extensions` (not a `firebase/` repo), and it requires a separate Twilio account + billing. It is not Firebase-native. It is the most mature and well-documented of the SMS extensions.

### (c) Build-Your-Own (Firebase Cloud Functions + Direct SDK)

Any carrier can be integrated directly via Cloud Functions:
- Write an HTTP-triggered or Firestore-triggered Cloud Function
- Import the carrier's npm SDK (Twilio, MSG91, Vonage, Plivo, etc.)
- Call the carrier's API to send the SMS

This is functionally identical to what the Extensions do internally — Extensions are just pre-packaged Cloud Functions.

## Section 4 — User's Likely Interpretations of "Firebase Inbuilt SMS We Have"

### (i) Firebase Phone Auth OTP Free Quota — REAL, but restricted to verification codes only

Firebase Auth does send real SMS to real phone numbers as part of `signInWithPhoneNumber`. It has a free quota (10,000 SMS/month per project). However:
- The SMS content is always a verification code generated by Firebase (e.g., "Your verification code is 123456").
- There is no API parameter to set custom message content.
- This cannot be used to send "Your therapist is on the way" type messages.
- Source: https://firebase.google.com/docs/auth/web/phone-auth

### (ii) Firebase Cloud Messaging (FCM) — NOT SMS

FCM is a push notification service for mobile/web apps. It sends messages to device tokens (app instances), not to phone numbers. FCM does not touch the carrier network. If a user has your app installed and a notification token, FCM can push to them. If the user has no app/token, FCM cannot reach them. SMS and FCM are completely separate channels.
- Source: https://firebase.google.com/docs/cloud-messaging/

### (iii) Identity Platform MFA SMS — REAL, but restricted to 2FA verification codes only

Identity Platform (Firebase's enterprise-tier auth) supports SMS as a second factor during login. Firebase sends the 2FA code via SMS. However:
- Message content is always an auth code — not customizable for transactional content.
- This is a sign-in flow feature, not a messaging API.
- Source: https://docs.cloud.google.com/identity-platform/docs/web/mfa

### (iv) Misconception: Admin SDK `sendSMS()` — DOES NOT EXIST

There is no `sendSMS()`, `sendMessage()` (for phone numbers), or any SMS method in the Firebase Admin SDK for any language (Node.js, Python, Java, Go, C#). The `admin.messaging()` namespace in the Admin SDK is exclusively for FCM push notifications. This method has never existed and is not on Firebase's public roadmap.
- Evidence: https://firebase.google.com/docs/reference/admin/node/firebase-admin.messaging (no SMS methods listed)
- GitHub search for feature requests on firebase-admin-node returns no `sendSMS` issues — the community consistently redirects SMS needs to Twilio/Vonage extensions.

## Section 5 — Honest Recommendation for Glamornate

Glamornate needs to send transactional SMS ("Your therapist is on the way", booking confirmations, cancellation alerts). None of Firebase's inbuilt features support this use case. The options are:

### Option A — Keep MSG91 Firebase Extension (Current Architecture)

**What it is:** MSG91's extension on extensions.dev uses a Firestore-trigger pattern. A Cloud Function watches a Firestore collection; when a document is written, the function calls MSG91's API to send the SMS.

**Pros:**
- Already implemented in Glamornate (the Firestore-trigger pattern exists in the codebase).
- MSG91 is India-focused with competitive rates for Indian phone numbers — appropriate for Glamornate's target market.
- No npm SMS SDK in Cloud Functions source — follows the Firestore-trigger architecture pattern.
- `firebase ext:install msg91/msg91-send-msg` is the one-time setup step.

**Cons:**
- MSG91 is a third-party company — not Firebase/Google. This is unavoidable for any SMS solution.
- Community-maintained extension (not Firebase-maintained). Breaking changes in MSG91's API would require waiting for the extension maintainer to update.
- Requires a MSG91 account and API credentials.

**Verdict: Recommended for Glamornate.** MSG91 is the most appropriate carrier for an India-focused spa booking platform. The Firestore-trigger pattern is already in place.

---

### Option B — Switch to Twilio Firebase Extension

**What it is:** `twilio/send-message` on extensions.dev. Same Firestore-trigger pattern, different carrier (Twilio).

**Pros:**
- Twilio is the largest SMS carrier globally; most mature extension; actively maintained by Twilio Labs.
- Better documentation and community support.
- Supports WhatsApp as well as SMS from the same extension.

**Cons:**
- Twilio has higher per-SMS costs for Indian numbers compared to MSG91.
- Still requires a third-party Twilio account — not more "Firebase native" than MSG91.
- The user has explicitly stated "Not Twilio."

**Verdict: Not recommended** — violates the explicit user constraint, and MSG91 is a better fit for India pricing.

---

### Option C — Defer SMS, Use FCM + Email Fallback

**What it is:** Use FCM push notifications (truly Firebase-native) for users with the app installed, and SendGrid email for those without. No SMS at all.

**Pros:**
- FCM is genuinely Firebase-native.
- No carrier costs or third-party SMS accounts needed.
- Simpler operational footprint.

**Cons:**
- FCM requires the user to have the app installed and notifications enabled. Users without the app or with notifications disabled receive nothing.
- SMS has dramatically higher open rates and is more reliable for time-sensitive notifications (therapist en route, appointment imminent).
- Degrades the product experience significantly for a service like Glamornate where "therapist is 5 minutes away" is time-critical.

**Verdict: Not recommended for Glamornate's primary notification channel.** Can be used as a supplementary channel alongside SMS, but should not replace it.

---

## Summary

Firebase has no inbuilt arbitrary SMS API. The closest things are:
1. Phone Auth OTP (verification codes only, not customizable)
2. Identity Platform MFA SMS (2FA codes only)
3. FCM (push notifications to devices, not SMS to phone numbers)

For Glamornate's transactional SMS needs, **MSG91 via Firebase Extension** (Option A) is the correct approach — it is the most Firebase-integrated available pattern for a non-Twilio carrier targeting Indian phone numbers, and it is already implemented in the codebase.
